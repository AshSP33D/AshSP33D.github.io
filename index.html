ASH'S PORTFOLIO

This project was tasked to me by my drum teacher. While initially developing it in Python, and writing a HTML file, I decided to switch over to constructing the page from within the browser using JavaScript for user convenience. It makes use of a custom font, results in 10, 16-beat bars and can be printed.
Being developed off the other drum sequences project, I simply removed some pieces of code I felt weren't needed any more (as I was creating my own sequences to go into the array) and replaced the sequences to replace with the Basic Rock variety. I decided to reduce the amount of checking the code had to do and the system I used for the arrays. In depth explanations can be found within the HTML files themselves.
I undertook this project myself with the sole intention of being able to host some of my small experiments on a Wordpress site (which I decided against once it became clear that the process was extremely inefficient. I quickly learned that you can only run JS, which is required, on premium Wordpress servers, which meant that I had to build the page on the only premium server I have access to. I felt it was a nice challenge, and could be used for other purposes by other people (for example for hosting a portfolio without having to purchase multiple different addresses). Of course, GitHub is still a better place to do the same function.
During work experience I was tasked with using C# to program robot AI that would participate within a fight. I learned the approaches that each in the development team took throughout the week and eventually applied the knowledge into the creation of a couple of robots at the end of the week. There was no documentation for the robots, so all understanding of the system was required to stem from the code itself. This is one robot that eliminates all other robots using an exploit found and pointed out to me by one of the developers in the code as well as integer underflows to set their scores to zero without being penalised.
This is another bot I developed similarly but with more complex AI. Despite being more complex than any of the other robots it was put up against (excluding my other bot) it repeatedly came out in fourth and fifth places. It mimics the behaviour of other example robots according to certain variables such as where it is in the leaderboard, how much health it has left etc.
I always choose to go for the harder option as long as it's more customisable. This was one of the reasons behind why I chose to program my portfolio from scratch. I felt my knowledge in HTML, CSS and JS was enough to create a useful and acceptable website, with more customisability being allowed than template websites such as Wordpress. Most of the code I initially used to create the barebones of the website came from an earlier project I did - a web-based launcher for retro games. Eventually most of the code was rewritten to fit better to this purpose but it allowed me to focus more on expandability and further customisability if needed. As this website is somewhat in use by other people (as a result of the drum sequences), I decided to have that as the first value in the table - immediately they know that they're on the right page.
