During work experience I was tasked with using C# to program robot AI that would participate within a fight. I learned the approaches that each in the development team took throughout the week and eventually applied the knowledge into the creation of a few robots by the end of the week. There was no documentation for the robots, so all understanding of the system was required to stem from the code itself.
This robot eliminates every other robot immediately as their round starts. It functions as the RobotWars project does not check for attacks of negative figures, meaning instead it will add health to a target robot - this was discovered by one of the developers. I then considered the use of integer underflows to break the system (at the time unknowing fully what the integer would be set to) to either set the target's health to an incredible negative figure or to zero. This means that it doesn't get penalised for the high amount of damage it does. It has a win rate over 10,000 tests of 100% in the unfair mode and 76% in the fair mode.
This is another bot I developed similarly but with more complex AI. Despite being more complex than any of the other robots it was put up against (excluding my other bot) it repeatedly came out in fourth and fifth places. It mimics the behaviour of other example robots according to certain variables such as where it is in the leaderboard, how much health it has left etc. It has a measly 3.9% win rate, despite being fairly complex.
This robot hit all available robots equally according to how many are alive at the time. If it has attacks left, it disregards them (thus making it somewhat inefficient) and finishes it's round. Due to this, and the strategy it undertakes, it gets a 0% win rate in tests.
This was the first robot made. Originally, it adapted code from one of the existing bots and targetted competitors by alphabetical order. This was somewhat effective, giving it a consecutive 2nd place. However, it was configured to mainly work only with the example bots, and required the most powerful robot (called Bully Robot) to be first in alphabetical order, meaning it became inefficient when placing it against other bots. Despite this, later in the week, a chance was given to improve the bot. I improved the logic of the bot but this lowered the efficiency of the bot. The developed version can be seen in the download. It still focusses on prior knowledge of the names of the robots however.
I always choose to go for the harder option as long as it's more customisable. This was one of the reasons behind why I chose to program my portfolio from scratch. I felt my knowledge in HTML, CSS and JS was enough to create a useful and acceptable website, with more customisability being allowed than template websites such as Wordpress. Most of the code I initially used to create the barebones of the website came from an earlier project I did - a web-based launcher for retro games. Eventually most of the code was rewritten to fit better to this purpose but it allowed me to focus more on expandability and further customisability if needed. As this website is somewhat in use by other people (as a result of the drum sequences), I decided to have that as the first value in the table - immediately they know that they're on the right page. Included in the website is also a Pi calculator running in the background as soon as you open the website which you can find the current value to in console. Type 'fastPi()' into console to disable functionality of the website and turn your computer into a Pi-calculating powerhouse.
This website plays upon an inside joke between me and one of my school friends. It was originally a Python script outputting a HTML file and opening it automatically. I decided to port it over to JS however when I felt that the file became unreasonably large (6MB after 2 weeks) with an increasingly long loading time. This meant that I could incorporate all the code into it's own site however, allowing me to host it online. The page has a pre-defined start position in Unix time and calculates how many seconds has passed since then - this is represented by the word 'good' in the website. It also logs a number of different details in the console, such as the current Unix time, the 'goodness' and the load time of the website, just for my benefit really.
